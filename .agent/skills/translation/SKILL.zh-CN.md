# translation - 代码注释与日志一键翻译（中英对照）

面向工程实践的“一键翻译”技能：默认直接对目标目录执行 **扫描 → 翻译 → 质量验证 → 报告**，并且尽量做到：

- 只改注释与日志可读文本，不改代码语义
- **翻译为中文，同时保留对应英文原文（中英对照）**
- 保留常见技术术语为英文（例如 API/HTTP/JSON）
- 出现语法/质量问题时可回滚到备份并在报告中标记

## 用法

```bash
/translation [目标目录路径] [--mode=swarm|serial]
```

- `目标目录路径`：可选。默认为当前工作目录（命令执行路径）。
- `--mode`：可选。默认 `swarm`（并行），可选 `serial`（串行）。

示例：

```bash
/translation
/translation /path/to/project/src
/translation /path/to/project --mode=serial
```

## 默认行为（契约）

### 翻译范围

- 翻译对象：代码注释（单行/多行/文档注释）与日志语句中的自然语言文本。
- **中英对照保留**：翻译后**必须保留原始英文**，不得丢弃。具体规则：
  - 多行/块注释：在英文原文下方追加中文译文（或反之），二者并存。
  - 单行注释：采用 `原英文 | 中文译文` 的形式，或在上一行补一行中文注释，确保英文不被删除。
  - 日志字符串：默认仅保留英文（避免破坏日志检索与告警匹配）；如确需中文，使用 `English text (中文)` 形式追加，禁止替换原英文。
- 保持不变：标识符、函数名、类名、路径、URL、正则、格式化占位符（例如 `%s`、`{name}`）、代码片段与缩进布局。
- 技术术语：默认保持英文（可在会话中扩展术语表）。

### 默认跳过（避免误改与噪声）

默认不处理以下目录/文件模式（可在会话中调整）：

- 目录：`node_modules/`、`dist/`、`build/`、`vendor/`、`coverage/`、`.git/`
- 文件：`**/*.min.*`、大体积文件（例如 > 1MB）与明显的生成文件（例如 `generated/`、`__generated__/`）

### 输出与可追溯性

每次运行会在目标目录下创建（或复用）工作目录：`<目标目录>/.translation/`

最小输出集合：

- 会话状态：`<目标目录>/.translation/sessions/<session-id>.md`
- 执行日志：`<目标目录>/.translation/logs/<session-id>.log`
- 原始文件备份：`<目标目录>/.translation/backup/<session-id>/...`

## 子任务委派机制（DeerFlow）

> 重要：本技能在 DeerFlow 中运行，**没有 `.md` 形式的子 agent 文件**，也不能用 `file:///` 链接引用子 agent。
> DeerFlow 通过内置的 `task` 工具委派子任务，可用的 `subagent_type` 为 `general-purpose` 与 `bash`。
> 注意：子 agent 默认禁用 `task` 工具（不能再嵌套委派），因此**激活本技能的主 agent 充当协调器**，
> 由主 agent 负责拆分清单并发起子任务，不能让子 agent 之间互相调用。

调用方式（示意）：

```text
task(
  subagent_type="general-purpose",
  description="translate file batch",
  prompt="<明确的翻译范围、中英对照规则、目标文件清单、备份与写回要求>"
)
```

## 一键工作流

### swarm（默认，并行）

1. **协调（主 agent）**：初始化会话与状态文件（`<目标目录>/.translation/`）。
2. **扫描**：主 agent 用 `Glob`/`Grep` 或一个 `general-purpose` 子任务产出待翻译清单（含文件列表、注释/日志统计、优先级、推荐分批粒度）。
3. **翻译（并行子任务）**：主 agent 将清单按批拆分，**并行**发起多个 `task(subagent_type="general-purpose")`，每个子任务负责一批文件：
   - 写回前先把原文件复制到 `backup/<session-id>/` 备份。
   - 严格执行“中英对照保留”规则，仅改注释/日志文本。
4. **质量验证**：所有翻译完成后，主 agent 发起一个 `general-purpose` 子任务（或自行）执行：完整性检查（英文是否保留）、术语一致性、语法/可解析性检查（可用 `bash` 子任务跑 lint/编译/语法解析），产出质量结论。
5. **汇总报告**：主 agent 汇总成功/失败/需人工处理清单，并给出输出路径与回滚说明。

### serial（串行）

与 swarm 相同的扫描/翻译/验证步骤，但**逐文件串行**发起子任务，便于小项目或更强上下文一致性需求。

## 失败与回滚（关键保证）

满足任一条件时，该文件应被标记为失败并从 `backup/<session-id>/` 回滚（至少在报告中明确指出）：

- 翻译后出现明显语法错误或文件无法解析/编译。
- **原始英文被删除**（违反中英对照保留规则）。
- 关键技术术语被错误翻译且影响理解。
- 质量验证未通过（具体阈值由验证步骤策略决定）。

## 执行规则

- **不改语义**：只修改注释与日志中的自然语言文本，绝不改动代码逻辑、标识符与占位符。
- **必留英文**：任何情况下都不得删除原始英文，确保可回溯与日志检索不受影响。
- **先备份后写回**：写回任何文件前，必须先备份到 `backup/<session-id>/`。
- **幂等**：重复运行不应造成重复叠加翻译；可通过检测已存在的中文译文行跳过。
- **遵守委派约束**：仅主 agent 发起 `task`，子 agent 不再嵌套委派。
